    6. Chat GPT - "La principal diferencia entre getElementBy* y querySelector* en el DOM es la forma en que buscan elementos en el documento:

  6. ¿Qué es un evento?

    7. Chat GPT - "En el contexto del DOM, un evento es una acción que ocurre en una página web y que puede ser detectada y respondida por un programa. Ejemplos comunes de eventos en el DOM incluyen hacer clic en un botón, mover el mouse sobre un elemento, presionar una tecla en el teclado, o cargar la página.

    Permiten que los programas web respondan a las acciones del usuario y actualicen dinámicamente el contenido, estilo o estructura de la página.

    Se pueden detectar con métodos (funciones) como 'addEventListener()', 'removeEventListener()', y event.target."

    developer.mozilla.org - The Event interface represents an event which takes place in the DOM.

    It can be triggered by the user action, like clicking the mouse button or tapping keyboard, or generated by APIs to represent the progress of an asynchronous task; as well as by being triggered programmatically.

    For example, by calling the 'HTMLElement.click()' method of an element, or by defining the event, then sending it to a specified target using EventTarget.dispatchEvent().

    There are many types of events, some of which use other interfaces based on the main Event interface. Event itself contains the properties and methods which are common to all events.

  8. ¿Qué es un callback?

    9. Chat GPT - "Esta funcion de retorno que se pasa como argumento a otra función y que se invoca en un momento posterior, generalmente después de que se haya completado alguna operación asíncrona.

    Éstos (callbacks) son una forma común de manejar eventos asíncronos en JavaScript, como solicitudes de red o animaciones. Al pasar una función como callback, el código JavaScript puede manejar estas operaciones asíncronas sin bloquear la ejecución del resto del código!"

    developer.mozilla.org - Éstos son funciones que se pasan a otra función como argumento, que posteriormente se invoca dentro de la función externa para completar algún tipo de rutina o acción.

    Por otro lado, usualmente se usan los callbacks para continuar con la ejecución del código después de que se haya completado una operación a sincrónica: los "retornos de llamada asincrónicos".

Respuestas: Por un lado me llamó mucho la atención como en unos casos, como en el ejercicio 6 con la función de generar números aleatorios, eventualmente se creaban funciones ya conocidas en otros lenguajes de programación como C++ y ‘C’, pero en javascript. Respecto al uso del ChatGPT para completar estos ejercicios, para los códigos J.Script, me llamó mucho la atención que éste (Chat) planteaba una respuesta diferente cada vez que se lo volvía a preguntar. cuando cada ejercicio ya estaba completo, me impresionó mucho cómo un conjunto de código puede hacer realidad algo que en un inicio me pareció complejo de hacer; cómo mostrar la posición del cursor, o agregar una nueva fila o columna de una tabla.

En primera instancia, usaría GitHub Copilot para que complete líneas de código que empecé a escribir; o para pasar un código de un lenguaje de programación a otro, que tengo entendido es una de las características más interesantes de este software. Debo mencionar que todavía no tengo acceso a éste; por eso dije ‘en primera instancia’.

Adicionalmente, usaría ChatGPT cuando tenga preguntas concisas, inclusive puede ser si va a llover hoy o no; pero no lo usaría como una alternativa a ‘estudiar’ sobre un tema particular, y que luego me cuente un resumen del mismo.

Esto yo lo relacionaría con la acción de usar A.I. para programar. Por un lado, me parece muy interesante cómo este tipo de herramientas, hacen que nuestras tareas, en este caso, puedan hacerse de manera más fácil y se completen rápidamente gracias a su ayuda.

Sin embargo, esencialmente estarían completando y casi haciendo todo mi trabajo, cuando muchas veces la idea es que yo aprenda programando, inclusive si el código no se ejecuta de manera adecuada y hay errores en el mismo. Hay que tener claro que éstas (herramientas) no son del todo perfectas; como me pasó que con cada vez que le preguntaba al ChatGPT una misma pregunta varias veces, en cada una me respondía algo diferente que no siempre era una respuesta correcta.

En sí tanto el ChatGPT como el Copilot son modelos de Machine Learning, cuya data inicial no siempre está de todo actualizada o completamente pulida sin errores. No son del todo perfectas. Por lo cual no estaría bien confiar plenamente en éstos, cuando se trate de resolver tareas o preguntas complejas; porque uno creería todo lo que le respondan, y no necesariamente verificaría con otras fuentes que estén en lo cierto. Estas herramientas nos van a ayudar hacer las tareas un poco más fáciles de realizar; pero hay que tener claro que algunas veces, lo que nos respondan o propongan para ayudarnos no siempre será del todo cierto.
